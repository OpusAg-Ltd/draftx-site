• OpusAg never stores your drawings or CAD data. DraftX processes them locally on your machines. We hold no copy.

• The AI drawing sorting feature runs on AWS Bedrock in the London region (eu-west-2), brokered by OpusAg. Drawing images are processed in-memory only — nothing is stored, logged, or used to train models — see section 3.

• We collect the minimum needed to run licensing: contact details, licence keys, machine fingerprints and validation logs.

• We don't sell data, we don't advertise, and the app contains no tracking.

3. AI drawing sorting

DraftX includes an AI classification feature for sorting drawings, provided as a managed service. How it works and where the data goes:

  • When you run an AI sort, DraftX sends a rendered image of each drawing, plus its filename, OCR text and basic part metadata, to the OpusAg licence server, which forwards it to AWS Bedrock (Anthropic Claude models) for classification.
  • Pass-through only: the OpusAg server holds the request in memory for the duration of the call and forwards the model's answer back to your machine. We do not store, log, or retain drawing images, OCR text, or model responses — the only records kept are usage metadata (token counts per job) for quota and billing.
  • Data residency: inference runs in the AWS London region (eu-west-2) using an EU cross-region inference profile, so processing stays inside UK/EU AWS regions.
  • No training: AWS Bedrock does not retain prompts or completions and does not use them to train models, under AWS's service terms.
  • Engineering drawings can contain personal data in title blocks (e.g. "drawn by" names). For that data, your organisation is the controller; OpusAg Ltd and AWS act as processors, per the summary in section 6.
  • The feature can be disabled at any time, and DraftX falls back to non-AI sorting methods.

4. Where it's stored

The licence server is hosted on Railway's cloud platform. Installer downloads are served via GitHub Releases, which may log standard web-request data (such as IP addresses) under GitHub's own privacy policy.

5. How long we keep it

  • Licence and contact records: life of the licence plus 6 years.
  • Validation logs: 12 months, then deleted or anonymised.
  • Expired trial records: 12 months after expiry unless converted to a paid licence.
